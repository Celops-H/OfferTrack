---

## 1. 执行目标

本计划用于指导 OfferTrack 从 v1.0 重构到 v2.0。

重构目标：

1. 新增面试组，支持按求职批次组织记录。
2. 将固定阶段重构为全局流程节点模板 + 记录内节点快照。
3. 支持任意节点通过、终止、撤回。
4. 删除面经链接和技术面自评分。
5. 看板改为动态列并全局展示所有组记录。
6. 日历展示具体时间并支持 hover 查看日期详情。
7. 增加数据导入导出。
8. 增加本地服务模式和应用内关闭按钮。

---

## 4. 关键依赖关系

```text
类型模型重构
  └── Store 重构
        ├── 设置页
        ├── 记录列表
        ├── 详情页
        ├── 看板
        ├── 日历
        └── 导入导出

本地服务与关闭应用
  └── 依赖 build 产物，但可在 UI 完成后独立接入
```

说明：

- 阶段 1 和阶段 2 是所有 UI 改造的前置条件。
- 阶段 4 设置页依赖节点模板 Store 能力。
- 阶段 5 记录列表依赖分组 Store 能力。
- 阶段 6 详情页依赖节点操作 Store 能力。
- 阶段 7 看板依赖模板和节点快照模型稳定。
- 阶段 8 导入导出应在数据模型稳定后做。
- 阶段 9 可相对独立，但关闭按钮需接入 Layout。

## 6. 风险控制

| 风险 | 表现 | 控制策略 |
|------|------|----------|
| 一次性改动过大 | 类型错误扩散，难以定位 | 先类型和 Store，再逐页迁移 |
| 旧数据迁移失败 | 用户历史记录丢失或异常 | 保守迁移，导入导出前置备份建议 |
| 模板快照复杂 | 看板动态列无法匹配旧模板 | 实现“其他流程”兜底列 |
| 分页筛选错乱 | 空页、页码越界 | 筛选条件变化时重置页码 |
| 关闭服务误触 | 用户误点关闭 | 关闭前弹出确认框 |
| 本地服务端口占用 | 启动失败 | server 输出提示，后续可扩展端口检测 |
