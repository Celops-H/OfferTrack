## 1. 产品概述

### 1.1 产品定位

OfferTrack 是一款面向密集求职期的**个人面试进度管理工具**，帮助用户按求职批次管理面试记录、追踪每家公司的流程节点、记录面试安排，并通过看板和日历获得全局视角。

### 1.2 目标用户

处于密集求职期、同时跟进多家公司、多批次面试流程的求职者。

典型场景：

- 同时记录“26年日常实习”“27年暑期”“秋招提前批”等不同批次。
- 不同公司流程差异较大，需要灵活配置流程节点。
- 希望用看板快速查看当前卡在哪些流程节点。
- 希望用日历查看各轮面试的具体时间。
- 希望可以导出数据作为备份。

### 1.3 核心价值

- **批次管理**：通过面试组组织不同求职周期的记录。
- **灵活流程**：通过全局流程节点模板适配不同招聘流程。
- **进度追踪**：每条记录以节点快照形式记录流程状态，支持任意节点终止。
- **全局可视化**：看板展示所有组的进行中记录，日历展示所有面试安排。
- **本地可控**：数据保存在本地浏览器存储，支持 JSON 导入导出。
- **轻量启动关闭**：无需 Electron，通过本地服务和应用内按钮关闭进程。

### 1.4 使用平台

- 桌面端 Web。
- 优先面向 Windows 本地使用。
- 移动端专项适配暂不考虑。

## 2. 核心概念定义

### 2.1 面试组（Group）

面试组用于按求职批次归类记录，例如：

- 26年日常实习
- 27年暑期
- 秋招提前批

规则：

- 每条记录必须属于一个面试组。
- 系统初始化时自动创建“默认分组”。
- 历史记录无 `groupId` 时自动迁移到默认分组。
- 默认分组不允许删除。
- 组内存在记录时不允许删除该组。

字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 组唯一标识 |
| name | 文本 | 是 | 组名 |
| createdAt | 时间戳 | 自动 | 创建时间 |
| updatedAt | 时间戳 | 自动 | 最近更新时间 |

---

### 2.2 公司面试记录（InterviewRecord）

一条公司记录代表用户与某家公司某个岗位的一次完整面试流程。

规则：

- 一条记录必须属于一个面试组。
- 一条记录创建时，会从当前全局流程节点模板生成一份流程节点快照。
- 后续修改全局模板不影响已有记录。
- 一条记录可添加多个面试轮次。

字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 记录唯一标识 |
| groupId | UUID | 是 | 所属面试组 |
| companyName | 文本 | 是 | 公司名 |
| position | 文本 | 是 | 岗位名称 |
| stageNodes | StageNode[] | 是 | 流程节点快照 |
| isEnded | 布尔 | 是 | 是否结束 |
| endReason | 枚举 | 否 | `completed` 或 `terminated` |
| rounds | Round[] | 是 | 面试轮次列表 |
| createdAt | 时间戳 | 自动 | 创建时间 |
| updatedAt | 时间戳 | 自动 | 最近更新时间 |

本版本删除以下字段：

- 面经链接 `interviewDocUrl`
- 技术面自评分 `selfRating`

---

### 2.3 全局流程节点模板（StageTemplate）

全局流程节点模板定义新建记录默认拥有的流程节点。

默认模板：

| 顺序 | 节点名称 |
|------|----------|
| 1 | 简历筛选 |
| 2 | 测评 |
| 3 | 技术面 |
| 4 | HR面 |
| 5 | Offer |

规则：

- 用户在设置页统一维护模板。
- 可以新增、重命名、删除、排序模板节点。
- 至少保留 1 个模板节点。
- 模板变更只影响后续新建记录，不影响已有记录。

字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 模板节点唯一标识 |
| name | 文本 | 是 | 节点名称 |
| order | 数字 | 是 | 排序 |
| createdAt | 时间戳 | 自动 | 创建时间 |
| updatedAt | 时间戳 | 自动 | 最近更新时间 |

---

### 2.4 记录流程节点（StageNode）

记录流程节点是从全局模板复制到具体记录中的节点快照。

字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 记录内节点唯一标识 |
| templateId | UUID | 是 | 来源模板 id |
| name | 文本 | 是 | 节点名称快照 |
| order | 数字 | 是 | 节点顺序快照 |
| status | 枚举 | 是 | 节点状态 |

节点状态：

| 状态 | 含义 | 颜色 |
|------|------|------|
| pending | 未开始 / 未到达 | 灰色 |
| ongoing | 进行中 | 蓝色 |
| passed | 已通过 | 绿色 |
| terminated | 流程在此终止 | 红色 |

状态规则：

- 新建记录时，第一个节点为 `ongoing`，其余为 `pending`。
- 一条记录最多只能有一个 `ongoing` 节点。
- 当前节点通过后变为 `passed`，下一个节点变为 `ongoing`。
- 最后一个节点通过后，记录视为已完成。
- 任意 `ongoing` 节点可以被标记为 `terminated`。
- 出现 `terminated` 节点后，记录视为已终止。
- 可以撤回到已通过或已终止节点，撤回后该节点变为 `ongoing`，后续节点变为 `pending`。

---

### 2.5 面试轮次（Round）

每条记录下可添加多个面试轮次，用于记录具体笔试、技术面、HR 面等安排。

字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 轮次唯一标识 |
| type | 枚举 | 是 | 笔试 / 技术面 / HR面 / 其他 |
| techRoundNumber | 数字 | 条件必填 | 仅技术面时填写 |
| format | 枚举 | 是 | 视频 / 现场 / 电话 |
| scheduledAt | 日期时间 | 是 | 面试时间 |
| duration | 数字 | 否 | 时长，单位分钟 |

本版本不再提供自评分字段。

## 3. 信息架构

```text
OfferTrack
├── 记录列表（主页）
│   ├── 面试组 Tab
│   ├── 组管理
│   ├── 组内记录分页
│   └── 公司详情页
├── 看板视图
│   └── 全局动态流程列
├── 日历视图
│   └── 全局面试日程
└── 设置页
    └── 全局流程节点模板管理
```

路由：

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 记录列表 | 主页，按组管理记录 |
| `/board` | 看板视图 | 全局展示所有组进行中记录 |
| `/calendar` | 日历视图 | 展示所有面试安排 |
| `/settings` | 设置页 | 管理全局流程节点模板 |
| `/record/:id` | 记录详情 | 单条记录详情和流程操作 |

## 4. 页面需求

### 4.1 记录列表页

功能：

- 展示面试组 Tab。
- 提供“全部”Tab。
- 支持新增、重命名、删除空组。
- 支持新增记录。
- 支持删除记录。
- 支持搜索公司名。
- 支持按状态筛选。
- 支持按最近更新时间倒序排序。
- 支持分页展示。

分页规则：

- 默认每页 10 条。
- 支持 10 / 20 / 50 条切换。
- 搜索、筛选、排序先执行，再分页。
- 切换组、搜索关键字、筛选条件时页码重置为第 1 页。
- “全部”Tab 同样分页。

新增记录规则：

- 当前在具体组 Tab 时，新记录默认归入当前组。
- 当前在“全部”Tab 时，新增弹窗中必须选择所属组。
- 新记录创建时使用当前流程模板生成节点快照。

---

### 4.2 记录详情页

功能：

- 展示和编辑公司名。
- 展示和编辑岗位名。
- 展示所属组，并支持修改所属组。
- 展示流程节点快照。
- 支持通过当前节点。
- 支持在当前节点终止流程。
- 支持撤回到已通过或已终止节点。
- 支持添加、删除面试轮次。
- 按时间排序展示轮次。

不再展示：

- 面经链接。
- 技术面自评分。

---

### 4.3 看板视图

功能：

- 根据当前全局流程节点模板动态生成列。
- 全局展示所有组记录。
- 不提供组筛选。
- 仅展示进行中的记录。
- 已完成、已终止记录不展示。
- 每条记录展示公司名、岗位名、所属组名。
- 点击记录跳转详情页。

兜底规则：

- 如果记录当前 `ongoing` 节点对应模板已被删除，则展示在“其他流程”列。
- “其他流程”列仅在存在这类记录时显示。

---

### 4.4 日历视图

功能：

- 以月视图展示所有组的面试安排。
- 日期格子内展示事件：`公司名 · 轮次类型 · HH:mm`。
- 点击事件跳转记录详情页。
- 鼠标悬停日期时展示当天详细信息。

详情内容：

- 日期。
- 公司名。
- 岗位名。
- 所属组名。
- 轮次类型。
- 技术面轮次号（如有）。
- 面试时间。
- 面试形式。
- 面试时长（如有）。

---

### 4.5 设置页

功能：

- 展示全局流程节点模板列表。
- 新增模板节点。
- 重命名模板节点。
- 删除模板节点。
- 调整模板顺序。

限制：

- 节点名称不能为空。
- 至少保留 1 个节点模板。
- 所有模板变更只影响后续新建记录。

---

### 4.6 全局工具能力

#### 数据导出

- 提供导出按钮。
- 导出 `stageTemplates`、`groups`、`records`、版本号、导出时间。
- 文件名：`offer-track-export-YYYY-MM-DD.json`。

#### 数据导入

- 提供导入按钮。
- 仅支持 `.json` 文件。
- 导入前校验格式。
- 导入前二次确认。
- 导入后覆盖当前本地数据。

#### 关闭应用

- 提供“关闭应用”按钮。
- 点击后调用本地服务 `/api/shutdown`。
- 服务进程退出。
- 开发模式下接口不可用时给出提示。

## 5. 全局交互规则

| 规则 | 说明 |
|------|------|
| 分组 | 每条记录必须属于一个组 |
| 默认组 | 自动创建，不允许删除 |
| 删除组 | 仅允许删除空组 |
| 模板快照 | 新建记录复制当前模板，历史记录不受模板变更影响 |
| 节点状态 | 灰色未开始、蓝色进行中、绿色通过、红色终止 |
| 看板 | 全局展示所有组进行中记录 |
| 日历 | 全局展示所有组面试安排 |
| 数据导入 | 覆盖当前数据前必须二次确认 |

## 6. 验收标准

### 6.1 面试组

- 可以新增、重命名、删除空组。
- 默认分组不能删除。
- 有记录的组不能删除。
- 新增记录可以正确归入目标组。
- 组内分页正确工作。

### 6.2 流程节点

- 设置页可以维护模板。
- 新记录使用当前模板生成流程快照。
- 修改模板不影响已有记录。
- 详情页可以推进、终止、撤回节点。
- 节点颜色符合规则。

### 6.3 看板

- 看板列由模板动态生成。
- 看板展示所有组的进行中记录。
- 已结束记录不展示。
- 已删除模板对应的进行中记录展示在“其他流程”列。

### 6.4 日历

- 日历事件显示具体时间。
- Hover 日期展示当天详情。
- 点击事件跳转详情页。

### 6.5 删除项

- 页面不再展示面经链接。
- 添加和查看轮次时不再展示自评分。

### 6.6 导入导出与关闭

- 可以导出完整 JSON。
- 可以导入合法 JSON 并覆盖数据。
- 可以通过应用内按钮关闭本地服务。

## 7. 暂不做事项

- Electron / Tauri 桌面应用打包。
- 文件系统数据库或 SQLite 存储。
- 云同步。
- 多用户协作。
- 移动端专项适配。
- 面试提醒 / 系统通知。
