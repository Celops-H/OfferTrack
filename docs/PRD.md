## 1. 产品概述

### 1.1 产品定位

OfferTrack 是一款 **Electron 桌面应用**，面向密集求职期的个人面试进度管理工具。用户按求职批次管理面试记录、自由定制每家公司的面试流程节点、通过看板和日历获得全局视角。

### 1.2 目标用户

同时跟进多家公司、多批次面试流程的求职者。

典型场景：

- 按"26年日常实习""27年暑期""秋招提前批"等批次组织记录。
- 不同公司流程差异大，需灵活配置流程节点（如某些公司有测评、某些没有）。
- 通过看板快速查看各节点卡了哪些公司。
- 通过日历查看各节点排期。

### 1.3 核心价值

- **批次管理**：面试组将记录按求职周期归类。
- **灵活流程**：全局流程节点模板，每条记录自由组合节点构成面试流程。
- **进度追踪**：节点快照记录状态，支持任意节点终止与回退。
- **排期可视化**：看板按节点分组，日历展示排期，hover 查看详情。
- **本地可控**：localStorage 存储 + JSON 导入导出。
- **桌面应用**：Electron 独立窗口，一键启动。

### 1.4 使用平台

- Windows 桌面应用（Electron）。

## 2. 核心概念定义

### 2.1 面试组（Group）

按求职批次归类记录。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 唯一标识 |
| name | 文本 | 是 | 组名 |
| createdAt | 时间戳 | 自动 | 创建时间 |
| updatedAt | 时间戳 | 自动 | 最近更新时间 |

规则：

- 每条记录必须属于一个面试组。
- 系统初始化时自动创建"默认分组"。
- 历史记录无 `groupId` 时自动迁移到默认分组。
- 默认分组不允许删除。
- 组内有记录时不允许删除该组。

---

### 2.2 全局流程节点模板（StageTemplate）

系统中可用的流程节点类型，所有面试组共享。用户在设置页统一维护。

默认模板：

| 顺序 | 节点名称 |
|------|----------|
| 1 | 简历筛选 |
| 2 | 测评 |
| 3 | 技术面 |
| 4 | HR面 |
| 5 | Offer |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 模板节点唯一标识 |
| name | 文本 | 是 | 节点名称 |
| order | 数字 | 是 | 排序 |
| createdAt | 时间戳 | 自动 | 创建时间 |
| updatedAt | 时间戳 | 自动 | 最近更新时间 |

规则：

- 可新增、重命名、删除、排序模板节点。
- 至少保留 1 个模板节点。
- 模板变更只影响后续新建记录，不影响已有记录。

---

### 2.3 记录流程节点（StageNode）

记录创建时从全局模板选择节点并自由排序，形成该记录的流程快照。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 节点唯一标识 |
| templateId | UUID | 是 | 来源模板 id |
| name | 文本 | 是 | 节点名称快照 |
| order | 数字 | 是 | 节点顺序 |
| status | 枚举 | 是 | 节点状态 |
| scheduledAt | 日期时间 | 否 | 排期时间（用于日历） |
| format | 枚举 | 否 | 面试形式：视频 / 现场 / 电话 |
| duration | 数字 | 否 | 时长（分钟） |

节点状态（StageNodeStatus）：

| 状态 | 含义 | 圆点颜色 |
|------|------|----------|
| `pending` | 未到达 | 灰色 |
| `ongoing` | 进行中 | 蓝色 |
| `passed` | 已通过 | 绿色 |
| `terminated` | 流程在此终止 | 红色 |

状态规则：

- 新建记录时，第一个节点为 `ongoing`，其余为 `pending`。
- 一条记录最多一个 `ongoing` 节点。
- 当前节点通过后变为 `passed`，下一节点变为 `ongoing`。
- 无下一节点时全部通过，流程结束。
- 任意 `ongoing` 节点可被标记为 `terminated`。
- 出现 `terminated` 后，记录视为已终止，后续节点保持 `pending`。
- 可点击已通过节点撤回，点击已终止节点恢复。

停留节点（当前/最后有意义的节点）下方文字：

| 场景 | 圆点颜色 | 下方文字 |
|------|----------|----------|
| 流程进行中 | 蓝色 | 进行中 |
| 流程终止 | 红色 | 流程终止 |
| 全部通过 | 绿色 | 流程结束 |

---

### 2.4 公司面试记录（InterviewRecord）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 记录唯一标识 |
| groupId | UUID | 是 | 所属面试组 |
| companyName | 文本 | 是 | 公司名 |
| position | 文本 | 是 | 岗位名称 |
| stageNodes | StageNode[] | 是 | 流程节点快照 |
| isEnded | 布尔 | 是 | 是否已结束 |
| endReason | 枚举 | 否 | `'completed'` 或 `'terminated'` |
| createdAt | 时间戳 | 自动 | 创建时间 |
| updatedAt | 时间戳 | 自动 | 最近更新时间 |

**v1.0 中删除的字段：**

- `stage` / `endStatus`（固定阶段枚举，替换为 stageNodes）
- `interviewDocUrl`（面经链接）
- `rounds: Round[]`（面试轮次，替换为 StageNode 上的排期字段）

## 3. 信息架构与路由

```
OfferTrack
├── 面试组列表（主页 /）
│   └── → 组内记录列表（/group/:groupId）
│       └── → 记录详情（/record/:id）
├── 看板（/board）
├── 日历（/calendar）
└── 设置（/settings）
```

## 4. 页面需求

### 4.1 面试组列表页（主页 `/`）

- 卡片网格展示所有面试组（组名 + 记录数）。
- 支持新增、重命名、删除空组。
- 默认组不可删除，非空组不可删除。
- 点击卡片进入该组记录列表。
- 无组时空状态引导。

### 4.2 组内记录列表页（`/group/:groupId`）

- 顶部显示组名 + 返回按钮。
- 新增记录弹窗：自动归入当前组，从模板勾选节点并排序，生成流程快照。
- 支持删除记录（二次确认）。
- 搜索公司名、按状态筛选（全部 / 进行中 / 已结束）、按更新时间倒序。
- 分页：默认 10 条，支持 10/20/50，搜索/筛选时重置页码。
- 记录卡片：公司名、岗位、流程进度条、所属组名、更新时间。
- 已结束记录卡片半透明。

### 4.3 记录详情页（`/record/:id`）

**基本信息区：**

- 公司名、岗位名行内编辑。
- 所属组下拉切换。

**流程节点区：**

- 节点进度条：节点名在圆点上方，圆点──圆点连接，状态文字在当前节点下方。
- 只有 `ongoing` 节点显示 [通过] [终止] 按钮。
- 点击已通过的绿色节点：弹确认 → 撤回到此节点（该节点 → ongoing，后续 → pending）。
- 点击已终止的红色节点：弹确认 → 恢复到此节点（该节点 → ongoing）。
- 灰色 `pending` 节点：无操作。

**节点排期区：**

- `ongoing` 或 `passed` 节点下可设置排期（日期时间、面试形式、时长），用于日历展示。
- 排期设置折叠在节点内或节点旁，非必填。

**不再展示：**

- 面经链接。
- 技术面自评分。
- 面试轮次列表（Round）。

### 4.4 看板视图（`/board`）

- 顶部面试组筛选：[全部] [组1] [组2] ...，默认"全部"。
- 列由当前全局模板节点动态生成。
- 每条记录展示在其 `ongoing` 节点对应的列中。
- 记录卡片：公司名、岗位名、所属组名（"全部"视图时）。
- 仅展示进行中记录，已结束记录不展示。
- "其他流程"兜底列：记录引用的模板已被删除时显示。
- 空列显示"暂无"。

### 4.5 日历视图（`/calendar`）

- 月历视图，数据来源为所有记录中有 `scheduledAt` 的 StageNode。
- 事件展示：`公司名 · 节点名 · HH:mm`。
- Hover 日期弹出 Popover 展示当天全部事件的详细信息：日期、公司名、岗位、所属组、节点名、面试形式、时长。
- 点击事件跳转记录详情页。

### 4.6 设置页（`/settings`）

- 全局流程节点模板的 CRUD + 排序。
- 至少保留 1 个模板节点。
- 删除时提示使用该模板的记录数。
- 提示："模板变更仅影响后续新建记录，不影响已有记录。"

### 4.7 全局工具

**导出：** 导航栏按钮 → Electron 原生保存对话框 → 导出 JSON 文件（`offer-track-export-YYYY-MM-DD.json`）。

**导入：** 导航栏按钮 → Electron 原生打开对话框 → 选择 JSON 文件 → 格式校验 → 显示摘要（组数/记录数/模板数） → 二次确认覆盖 → 刷新页面。

**导出格式：**

```json
{
  "version": "2.0",
  "exportedAt": "...",
  "stageTemplates": [...],
  "groups": [...],
  "records": [...]
}
```

## 5. 验收标准

### 5.3 看板
- [ ] 列由模板动态生成
- [ ] 面试组筛选正常工作
- [ ] "其他流程"兜底列在需要时显示
- [ ] 已结束记录不展示

### 5.4 日历
- [ ] 事件显示节点名和时间
- [ ] Hover Popover 详情完整
- [ ] 点击事件跳转详情页

### 5.5 删除项
- [ ] 面经链接、自评分、Round 轮次全部移除

### 5.6 导入导出
- [ ] 可导出完整 JSON，导入合法 JSON
- [ ] 导入前二次确认，非法数据不覆盖

- [ ] `npm run dev` 启动 Electron 开发窗口
- [ ] `npm run build` 构建可分发安装包
- [ ] 应用内可关闭窗口
