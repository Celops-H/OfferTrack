## 1. 开发阶段划分

整体开发分为 **5 个阶段**，按依赖关系串行推进。每个阶段完成后进行验收，确认无误再进入下一阶段。

```
阶段 1：项目初始化 + 基础骨架
    ↓
阶段 2：数据层（类型定义 + 状态管理）
    ↓
阶段 3：核心页面（记录列表 + 公司详情页）
    ↓
阶段 4：辅助视图（看板 + 日历）
    ↓
阶段 5：收尾（归档筛选 + 启动脚本 + 样式打磨）
```

---

## 2. 各阶段详细任务

### 阶段 1：项目初始化 + 基础骨架

**目标：** 项目能跑起来，有导航框架，点击能切换空白页面。

**验收标准：** 浏览器打开后能看到顶部导航，点击「记录列表」「看板」「日历」能切换对应空白页面。

### 阶段 2：数据层

**目标：** 类型定义完成，Zustand store 就绪，数据可持久化到 localStorage。

| # | 任务 | 产出 |
|---|------|------|
| 2.1 | 定义所有 TypeScript 类型和枚举（Stage、EndStatus、RoundType、InterviewFormat、Round、InterviewRecord） | `types/index.ts` |
| 2.2 | 实现 Zustand store，包含 CRUD 操作 | `store/useRecordStore.ts` |
| 2.3 | 配置 Zustand persist 中间件，绑定 localStorage | 同上 |
| 2.4 | 编写 store 的操作方法 | 同上 |

**store 需包含的操作：**

```
addRecord(record)           — 新增记录
deleteRecord(id)            — 删除记录
updateRecord(id, fields)    — 更新记录（公司名、岗位、阶段、面经链接等）
updateStage(id, stage, endStatus?)  — 推进阶段
addRound(recordId, round)   — 添加轮次
deleteRound(recordId, roundId)  — 删除轮次
getRecordById(id)           — 获取单条记录
```

**验收标准：** 在浏览器控制台中可以调用 store 方法增删改查数据，刷新页面数据不丢失。

## 3. 任务依赖关系

```
1.1 → 1.2 → 1.3 → 1.4 → 1.5 → 1.6
                                  ↓
                    2.1 → 2.2 → 2.3 → 2.4
                                        ↓
                          ┌─────────────┤

                    3.1 ~ 3.9     3.10 ~ 3.16
                    (记录列表)      (详情页)
                          ↓             ↓
                          └──────┬──────┘
                                 ↓
                    ┌────────────┤
                    ↓            ↓
              4.1 ~ 4.4    4.5 ~ 4.9
               (看板)        (日历)
                    ↓            ↓
                    └─────┬──────┘
                          ↓
                    5.1 ~ 5.6
                     (收尾)
```

- 阶段 1 → 2 → 3 严格串行
- 阶段 3 内部：记录列表（3a）和详情页（3b）可并行，但建议先完成 3a
- 阶段 4 内部：看板（4a）和日历（4b）可并行
- 阶段 5 在前 4 个阶段全部完成后进行

## 4. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Ant Design Calendar 自定义渲染不够灵活 | 日历视图可能无法按预期展示事件 | 预研 Calendar 的 `cellRender` API，必要时换用第三方日历库 |
| localStorage 数据意外丢失 | 所有面试记录丢失 | 收尾阶段后可追加 JSON 导入/导出功能 |
| 流程节点进度条组件实现复杂度 | 自定义组件开发耗时 | 使用 Ant Design Steps 组件简化实现 |

## 5. 开发规范

| 规范 | 说明 |
|------|------|
| 语言 | TypeScript strict 模式 |
| 组件 | 函数组件 + Hooks |
| 样式 | CSS Modules 或 Ant Design 内置样式，避免全局样式污染 |
| 命名 | 组件 PascalCase，文件与组件同名，工具函数 camelCase |
| 状态 | 全局状态用 Zustand，组件局部状态用 useState |
| 格式化 | 统一使用 Prettier（可选配置） |
