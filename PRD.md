## 1. 产品概述

### 1.1 产品定位

OfferTrack 是一款面向密集求职期的**个人面试进度管理工具**，帮助用户追踪与各公司的面试进程、记录每轮面试复盘，并从全局视角感知求职整体状态。

### 1.2 目标用户

处于密集求职期、同时跟进多家公司面试流程的求职者。

### 1.3 核心价值

- 全局可视化：看板和日历视图掌握整体求职状态
- 进度追踪：清晰记录每家公司的面试阶段和流转
- 面试复盘：结构化记录每轮面试信息，辅助自我改进

### 1.4 使用平台

桌面端（Web），移动端暂不考虑。

## 2. 核心概念定义

### 2.1 公司记录（Record）

一条公司记录代表用户与某家公司某个岗位的一次完整面试流程。

- 只记录有交互的公司（简历筛选通过、约面试等），不记录已投递但无回应的
- 一家公司同时只有一条进行中的记录
- 一条记录结束后，可为同一家公司新建新记录（重新投递新岗位）

**字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 公司名 | 文本 | 是 | |
| 岗位名称 | 文本 | 是 | 如「前端工程师」 |
| 当前阶段 | 枚举 | 是 | 见 2.2 阶段定义 |
| 面经外链 | URL | 否 | 外部文档链接（Notion/飞书/语雀等），点击跳转 |
| 创建时间 | 时间戳 | 自动 | |
| 最近更新时间 | 时间戳 | 自动 | |

### 2.2 阶段（Stage）

固定阶段，按以下顺序流转，允许跳过中间阶段：

```
简历筛选中 → 技术面 → HR面 → Offer阶段 → 已结束
```

「已结束」包含三种子状态：

| 子状态 | 含义 |
|--------|------|
| 已拿 Offer | 收到录用通知 |
| 挂 | 对方拒绝 |
| 已拒 | 用户主动拒绝对方（如拒掉 Offer） |

阶段流转在公司详情页内通过下拉选择手动操作。

### 2.3 面试轮次（Round）

每条公司记录下可添加多个面试轮次，用于记录每次面试的结构化信息。

**轮次类型：** 笔试 / 技术面 / HR面 / 其他

技术面支持多轮（第1轮、第2轮……），通过轮次号区分。

**字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 轮次类型 | 枚举 | 是 | 笔试 / 技术面 / HR面 / 其他 |
| 技术面轮次号 | 数字 | 条件必填 | 仅轮次类型为「技术面」时显示和填写 |
| 面试形式 | 枚举 | 是 | 视频 / 现场 / 电话 |
| 面试时间 | 日期+时间 | 是 | 用于日历视图展示 |
| 时长 | 数字（分钟） | 否 | |
| 自我评分 | 1-5 | 否 | 星级评分 |

## 3. 信息架构

```
OfferTrack
├── 记录列表（主页）           ← 增删记录、搜索、筛选、排序
│   └── 公司详情页             ← 阶段流转、面经外链、轮次管理
├── 看板视图                   ← 按阶段分布查看进行中的公司
└── 日历视图                   ← 按月查看面试安排
```

三个主视图通过顶部导航切换，公司详情页为子页面（从记录列表或看板点击进入）。

## 4. 页面详细设计

### 4.1 记录列表（主页）

#### 4.1.1 页面定位

应用主页面，承载记录的增删和浏览，展示所有公司记录（进行中 + 已结束）。

#### 4.1.2 顶部功能区

| 功能 | 说明 |
|------|------|
| 新增记录按钮 | 点击后弹出表单，填写公司名、岗位名称，默认阶段为「简历筛选中」 |
| 搜索框 | 按公司名模糊搜索 |
| 阶段筛选 | 下拉选择，选项包含所有阶段 + 已结束的三种子状态 |
| 排序 | 按最近更新时间排序（默认最新在前） |

#### 4.1.3 记录卡片

每条记录以卡片形式展示，包含以下元素：

```
┌──────────────────────────────────────────────────┐
│ 字节跳动 · 前端工程师                      [HR面]  │
│ ○ 简历筛选 ──● 技术面(3轮) ──● HR面 ── ○ Offer    │
│ 最近更新：2026-04-22                      [删除]   │
└──────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 公司名 + 岗位名称 | 卡片标题 |
| 当前阶段标签 | 右上角标签展示 |
| 流程节点进度条 | 已完成节点实心，当前节点高亮，未到节点空心 |
| 最近更新时间 | 底部左侧 |
| 删除按钮 | 底部右侧，点击需二次确认 |

**交互：**
- 点击卡片（非删除按钮区域）跳转到公司详情页
- 已结束的记录卡片整体标灰显示

---

### 4.2 看板视图

#### 4.2.1 页面定位

辅助视图，按阶段分布快速查看进行中的公司。

#### 4.2.2 布局

每个阶段一个模块卡片，模块内按行列出公司名：

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ 简历筛选中    │  │ 技术面        │  │ HR面          │
│  · Soul      │  │  · 美团       │  │  · 字节跳动    │
│  · 腾讯      │  │  · 快手       │  │               │
└──────────────┘  └──────────────┘  └──────────────┘

┌──────────────┐
│ Offer阶段     │
│  · 阿里       │
└──────────────┘
```

**规则：**
- 只展示进行中的记录，已结束的不出现
- 空阶段的模块仍然展示（显示为空）
- 点击公司名跳转到公司详情页

---

### 4.3 日历视图

#### 4.3.1 页面定位

以月视图日历形式展示面试安排时间线，涵盖历史和未来。

#### 4.3.2 布局

标准月历网格，日期格子内展示当天的面试事件：

| 元素 | 说明 |
|------|------|
| 事件标签 | 公司名 + 轮次类型（如「美团 · 技术面」） |
| 月份切换 | 左右箭头切换上/下月 |

**交互：**
- 点击事件标签跳转到对应公司详情页

---

### 4.4 公司详情页

#### 4.4.1 页面定位

单条公司记录的完整信息页，承载阶段流转、面经链接、轮次管理。

#### 4.4.2 页面结构

**基本信息区**

| 字段 | 交互 |
|------|------|
| 公司名 | 可编辑 |
| 岗位名称 | 可编辑 |
| 当前阶段 | 下拉选择，支持推进到任意阶段或标记结束（选子状态） |
| 面经链接 | 输入 URL，显示为可点击跳转的链接 |

**面试轮次区**

- 按面试时间从早到晚排列
- 每轮以折叠卡片展示，点击展开查看详情
- 页面底部「添加轮次」按钮，点击弹出轮次录入表单

**轮次卡片摘要展示：**

```
技术面 第2轮 | 视频 | 2026-04-20 14:00 | 60分钟 | ★★★★☆
```

## 5. 全局交互规则

| 规则 | 说明 |
|------|------|
| 阶段流转 | 仅在公司详情页操作，支持跳过中间阶段 |
| 删除记录 | 仅在记录列表卡片上操作，需二次确认 |
| 新增记录 | 仅在记录列表顶部操作 |
| 返回导航 | 详情页左上角返回按钮，返回来源页面 |
